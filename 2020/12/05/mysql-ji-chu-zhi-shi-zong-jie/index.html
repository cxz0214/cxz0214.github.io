第一章 了解SQL

1.1 数据库基础

  • 数据库(database)：以某种有组织的方式存储的数据集合.

  • 表(table): 某种特殊类型数据的结构化清单.

  • 模式(schema)：关于数据库和表的布局及特性的信息.

  • 列(column)：表中的一个字段

  • 数据类型(datatype)：所容许的数据的类型

  • 行(row)： 表中的一个记录

  • 主键（primary key）：一列（或一组列），其值能够区分表中每一行.

    • Mysql强制规则:

      • 任意两行都不具有相同的主键值
      • 每个行都必须具有一个主键值(主键列不允许为NULL值)

    • 主键的好习惯:

      • 不更新主键列的值
      • 不重用主键列的值
      • 不在主键列中使用可能会更改的值

1.2 什么是SQL

    SQL是结构化查询语言(Structured Query Language)的缩写，是一种专门用来与数据库通信的语言.

  • SQL的优点:
    • SQL不是某个特定数据库提供商专有的语言.
    • SQL简单易学.
    • SQL是一种强有力的语言，灵活使用其他语言元素，可以进行非常复杂和高级的数据库操作.

第三章 使用MySql

3.1 连接到mysql

  • 连接到mysql所需信息
    • 主机名
    • 端口
    • 一个合法的用户名
    • 用户口令

3.2 选择数据库

  • USE 数据库名:选择数据库，如 : USE mysqllearn;
  • 必须使用USE打开数据库才能访问其中的数据.

3.3 了解数据库和表

  • SHOW DATABASES:返回可用数据库的一个列表
  • SHOW TABLES:获得数据库内的表的列表.
  • SHOW COLUMNS FROM customers:显示表customers的列
  • DESCRIBE语句作为SHOW COLUMNS FROM 的一种快捷方式
  • SHOW STATUS:用于显示广泛的服务器状态信息.
  • SHOW CREATE DATABASE 数据库名:用于显示创建特定数据库的Mysql语句
  • SHOW CREATE TABLE 表名：用于显示创建特定表的mysql语句
  • SHOW GRANTS:用来显示授予用户(所有用户或特定用户)的安全权限
  • SHOW ERRORS和SHOW WARNINGS:用来显示服务器错误或警告信息

第四章 检索数据

4.1 检索单个列

  • 语法：SELECT 列名 FROM 表名;
  • 习惯:SQL语句不区分大小写，一般关键字用大写，列名或表名用小写

4.2 检索多个列

  • 语法：SELECT 列名1 , 列名2 ,… 列名n FROM 表名;

4.3 检索所有列

  • 语法：SELECT * FROM 表名;
  • 注意： 除非确定需要表中的所有列，斗则最好别用通配符*，因为这会降低检索和应用程序性能。

4.4 检索不同行

  • 语法：SELECT DISTINCT 列名1 , 列名2 , … 列名n FROM 表名;
  • 注意：不能部分使用DISTINCT关键字，DISTINCT关键字应用于所有列而不仅是前置它的列

4.5 限制结果

  • 语法：SELECT 列名(可多个) FROM 表名 LIMIT 开始行数 结果所需显示的行数;
  • 等价语法(Mysql5):SELECT 列名(可多个) FROM 表名 LIMIT 结果所需显示的行数 OFFSET 开始行数
  • 注意:行0是第一行，行1是第二行

4.6 使用完全限定的表名

  • 语法：SELECT 表名.列名 FROM 数据库名.表名;

第五章 排序检索数据

5.1 按一列排序

  • 语法：SELECT 列名(一或多个) FROM 表名 ORDER BY 排序列名;
  • 注意：排序列可以是选中的列，也可以是未显示的列

5.2 按多个列排序

  • 语法 :SELECT 列名(一个或多个) FROM 表名 ORDER BY 排序列名1 , 排序列名2，….;
  • 注意：按多个列排序时，排序完全按照所规定的顺序进行.

5.3 指定排序方向

  • 语法：SELECT 列名 FROM 表名 ORDER BY 列名 DESC(降序)/ASC(默认升序) ;
  • 注意:
    • DESC关键字只应用到直接位于其前面的列名.要对多个列降序必须对每个列指定DESC关键字.
    • ASC即升序是默认的，故无需设定.
    • 使用ORDER BY 与 LIMIT 组合可以找出一列中最大值或最小值，但是ORDER BY 必须位于 FROM之后 ，LIMIT 位于ORDER BY之后，次序不对将产生错误信息！

第六章 过滤数据

6.1 使用WHERE子句

  • 语法：SELECT 列名 FROM 表名 WHERE 条件;
  • 注意：WHERE 子句的位置应该位于ORDER BY之前.

6.2 WHERE 子句操作符

操作符 说明
= 等于
<> 不等于
!= 不等于
< 小于
<= 小于等于
> 大于
>= 大于等于
BETWEEN 在指定的两个值之间
  • 检查单个值
    • Mysql在执行匹配时不区分大小写
  • 范围值检查
    • BETWEEN 开始值 AND 结束值
  • 空值检查
    • 使用 WHERE … IS NULL;
    • 注意：NULL 无论是否匹配均不返回。

第七章 数据过滤

7.1 AND 操作符

  • 语法：SELECT 列名 FROM 表名 WHERE 条件1 AND 条件2;

7.2 OR操作符

  • 语法：SELECT 列名 FROM 表名 WHERE 条件1 OR 条件2;

  • 注意：OR 和 AND 同时出现在子句中时，AND优先级更高，可通过括号改变计算次序.

7.3 IN关键字

  • 语法：SELECT 列名 FROM 表名 WHERE 列名 IN (值1，值2) ;
  • 优点：
    • 在使用长的合法选项清单时，IN操作符的语法更清楚且更直观.
    • 在使用IN时，计算的次序更容易管理.
    • IN操作符一般比OR操作符清单执行更快.
    • IN的最大的优点就是可以包含其他SELECT语句,使得能够更动态地建立WHERE子句.

7.4 NOT操作符

  • 作用：WHERE 子句中用来否定后跟条件的关键字.
  • 注意：Mysql支持使用NOT对IN、BETWEEN和EXISTS子句取反，这与多数其他DBMS允许使用NOT对各种条件取反有很大的差别.

第八章 用通配符进行过滤

8.1 百分号（%）通配符

  • 说明：% 表示任何字符出现任意次数;
  • 注意：通配符可以在搜索模式中任意位置使用，并且可以使用多个通配符.
  • 例子：SELECT prod_id , pro_name FROM products WHERE pro_name LIKE ‘%anvil%’;
  • 注意：%代表搜索模式中给定位置的0个、1个或多个字符.

8.2 下划线(_)通配符

  • 说明：下划线（_） 只能匹配单个字符.
  • 例子：SELECT prod_id , prod_name FORM products WHERE prod_name LIKE ‘_ ton anvil’;

8.3 使用通配符的技巧

  • 不要过度使用通配符，如果其他操作符能达到相同的目的，应该使用其他操作符.
  • 在确实需要使用通配符时，除非绝对必要，否则不要把它们用在搜索模式的开始处。把通配符置于搜索模式的开始处，搜索起来是最慢的.
